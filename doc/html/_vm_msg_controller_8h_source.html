2 // This module defines a virtual machine messages controller
3 //
4 // All the messages that a virtual machine generates are controlled by this module. It has two main functionalites:
5 // - The messages has a commId parameter. This parameter is setted by the applications. For example, when an application is distributed
6 // it is used to identify the rank of the source/destination process. This module, storage the original value for this parameter
7 // and it is replaced with the position of the application at the gates vector. This is in order to know which is the owner of the message if
8 // it is responsed with a request.
9 // - The second functionality (under developement) is the migration manager. It control the quantity of messages that are out of
10 // application in order to block the out messages and wait for the incoming messages to disconnect gates and reconnect to the destination
11 // host when it would be possible
12 //
